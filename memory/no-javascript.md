Never suggest adding JavaScript to this project. The portfolio is pure HTML5/CSS3 with no build step and no framework.

**Why:** It is an intentional architectural constraint for the project — keeping it JS-free is a design goal, not an oversight.

**How to apply:** When solving interactivity/behavior needs, reach for HTML/CSS-only solutions; do not propose JS even as a fallback. Also recorded in `CLAUDE.md` (Conventions); mirrored here at the user's request.
